# Explanation

*Understanding-oriented. The reasoning behind the system — concepts, mechanisms,
and the *why* of design choices. Read these to understand, not to do a task.*

## Available

  layering, data flow.
- **[Data flow in detail](data-flow.md)** — a single worked trace from token to
  sound, with the real code touchpoints.
- **[How harmony works](how-harmony-works.md)** — SATB voicing, guide vs. color
  tones, the `pick_soprano` scorer, and a worked voicing example.
- **[How percussion works](how-percussion-works.md)** — interrupters, per-hit
  `prob`/`flam`, the two-tier randomness model, and a worked parse example.
- **[The token system](token-system-report.md)** — the synthesis claim and the
  provenance narrative.
- **[Music-theory companion](music-theory-companion.md)** — where the ideas sit in
  the field, with the reading list.
- **[Design decisions (ADRs)](decisions/index.md)** — one record per major choice (the *why*).
- **[Song studies](song-studies/kiss-on-my-list.md)** — literate walkthroughs of real pieces
  (e.g. [Kiss On My List](song-studies/kiss-on-my-list.md)).
