Music Generator

Write a chord progression as a line of text. The engine voices it into four-part harmony, adds percussion and melody, and hands you a MIDI file — or rendered audio.

How it works

The system is a pipeline from text to sound. You describe music in a compact notation; the engine does the rest — voice leading, rhythm, and rendering are all handled automatically.

1
Write tokens

Describe your progression, drums, and melody as compact tokens in the CLI or the web instrument.

↓
2
Parse & expand

Chord tokens resolve to pitch classes. Repetition groups like [A::min7, G]*8 expand into flat event lists.

↓
3
Voice & arrange

The harmony engine applies voice leading and voicing rules across soprano, alto, tenor, and bass. Percussion tokens become timed hits. Melody follows scale-degree rules.

↓
4
MIDI out

Events are assembled into a multi-track MIDI file. Optionally rendered to WAV/MP3 via FluidSynth and a SoundFont.

The token language

The heart of the project is a small notation. Each chord, drum hit, or melody note is a single token. A progression is a comma-separated sequence. The grammar is designed to be terse enough to type at the command line but expressive enough to describe complex harmony.

Chords

root[:inversion][:recipe][/bass][*reps]

C                  C major (bare triad)
C::maj7            C major seventh
A::min9            A minor ninth
A:1:min9           A minor ninth, 1st inversion
G::13              G dominant thirteenth
G::maj7/C          G major 7 over a C pedal bass
[A::min7, G]*8     repeat the group 8 times

Recipes cover common jazz and classical voicings — maj7, min9, 13, aug, dim7, mystic, messiaen_resonance, and dozens more.

Percussion

duration + instrument[modifiers]

qb               quarter-note kick
eg               eighth-note hi-hat
esc              eighth snare
qb,eg,esc,eg     a token-per-beat groove

# per-hit modifiers in brackets:
sb[vel+20]       sixteenth kick, louder
eg[prob.75]      hat fires 75% of the time

Duration letters: w whole · h half · q quarter · e eighth · s sixteenth. Instruments: b kick · c snare · g hat · i open hat · f clap · k ride.

Melody

duration + scale-degree

q1 q5 e4 e3 e2 e1   quarter and eighth notes
q7 h1               leading tone to tonic

# transforms apply to the whole line:
--melody-transform invert
--melody-transform transpose:5

Degrees are relative to the current key, so the same melody tokens work over any progression.

Full grammar reference: docs/reference/token-grammar.md

Generation modes

The same token language feeds four distinct compositional engines. Each mode shapes how the material unfolds over time.

Ostinato

A chord progression loops for as long as you specify. The workhorse mode — good for groove sketches, backing tracks, and quick experiments. 

--mode ostinato \
  --keys 'C::maj7, A::min9, D::min7, G::13' \
  --bars 32 --instrument epiano

Arrangement

A YAML song file defines named sections (verse, chorus, bridge) with their own keys, percussion, and instrument choices. Sections are assembled into a full song structure. 

--mode arrangement --song songs/kiss_on_my_list.yml

Fugue

A subject token sequence is harmonised and then developed through a classical fugal exposition — subject, answer, and stretto entries staggered across voices. 

--mode fugue \
  --melody 'q1 q5 e4 e3 q2 h1' \
  --melody-key C --bars 64

Process music

A short cell evolves by rule — additive (a note added each cycle, after Glass), phase shifting (two copies drift apart, after Reich), or augmentation. Minimal input, maximal unfolding. 

--mode process --process additive \
  --process-cell 'e1 e2 e3 e5 e7 e5 e3 e2' \
  --instrument organ --melody-key C

Demo library

Rendered from MIDI in your browser — no plugins, no SoundFont. Press play on any one. Regenerate them all with make gallery; the star demo is make demo.

★ Kiss On My List

arrangement · the demo button

The flagship. A full multi-section pop arrangement with a real sung lead (verse tune, pre-chorus lift, chorus hook) that turns to a bright saw on the choruses, over slash/pedal bass, a per-section tempo map, and a fill-rate arc that builds and releases — it plays like a record.

make demo          # or:
music_generator.py --song songs/kiss.yml --out kiss --no-play

Autumn Leaves

arrangement · walking bass

A sax head melody over walking bass and arpeggiated SATB on the ii–V–i cycle, jazz brushes, then a saw-lead solo chorus.

music_generator.py --song songs/autumn_leaves.yml --out autumn --no-play

A Whiter Shade of Pale

arrangement · descending bass

A Bach-style descending bass built entirely from slash chords (C, C/B, Am, Am/G, …) under church organ.

music_generator.py --song songs/whiter_shade_of_pale.yml --out wsop --no-play

Isn't She Lovely

arrangement · sung melody

A funk-soul groove with a real verse melody on top — a sung lead over octave bass and a syncopated kit, then a harmonica solo on the bridge.

music_generator.py --song songs/isnt_she_lovely.yml --out lovely --no-play

Dense colour voicing

dense · exotic recipes

The token DSL as a colour organ: extended and exotic chord sets (maj7#11, mystic, messiaen, quartal, 7alt) sounded complete across the register.

cook_song.py make dense_colors

Counterpoint study

counterpoint · suspensions

Four independent SATB lines over a ii–V cycle, with suspensions and anticipations woven in — voiced as a choir.

cook_song.py make counterpoint

Fugue exposition

fugal · subject & answer

Voices enter with subject then answer (up a fifth); the prior voice continues with the inverted countersubject, and a cadence closes it.

cook_song.py make fugue

Percussion evolution

staged grooves · fill curve

Over a held vamp the kit builds in four obvious stages — kick + hats → + backbeat snare → + off-beat open hats → full 16th-note kit with fills — while the fill rate ramps 0 → 0.6 across the piece.

cook_song.py make perc_evolution

Quick start

Clone & install 
git clone https://github.com/galenspikes/music-generator
cd music-generator
python3 -m venv venv && venv/bin/pip install -r requirements.txt
Generate MIDI 
venv/bin/python music_generator.py \
  --mode ostinato \
  --keys 'C::maj7, A::min9, D::min7, G::13' \
  --out my_song --no-play
Render audio (needs fluidsynth + ffmpeg) 
brew install fluidsynth ffmpeg   # macOS
./play_music --save-wav --sf2 path/to/soundfont.sf2 \
  --mode ostinato --keys 'C::maj7, A::min9, D::min7, G::13'
Or use the web instrument 
cd webapp && pip install -r requirements.txt
uvicorn backend:app --port 8753 &
cd frontend && npm install && npm run dev

Output lands in output/midi/, output/audio/, and output/metadata/. Full CLI reference: docs/reference/cli.md.
